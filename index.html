DDL in SQL

Struttura, vincoli e gestione dei database

Il DDL (Data Definition Language) serve per creare e modificare la struttura di un database.

  • CREATE DATABASE
  • CREATE TABLE
  • ALTER TABLE
  • DROP TABLE

CREATE e DROP DATABASE

Creano o eliminano un database.

CREATE DATABASE Scuola;
DROP DATABASE Scuola;

CREATE TABLE

Definisce struttura, campi e vincoli.

CREATE TABLE Studenti (
  ID INT PRIMARY KEY,
  Nome VARCHAR(50) NOT NULL,
  Eta INT CHECK (Eta >= 6)
);

ALTER TABLE e DROP TABLE

ALTER TABLE

ALTER TABLE Studenti
ADD Email VARCHAR(100);

DROP TABLE

DROP TABLE Studenti;

Modifiche avanzate

ALTER TABLE Studenti
DROP COLUMN Email;

ALTER TABLE Studenti
ADD CONSTRAINT chk_eta
CHECK (Eta >= 6 AND Eta <= 19);
Esercizio: aggiungere e rimuovere un campo.

Tipi di dato in SQL

Definiscono che valori può contenere una colonna

Numerici

INT, DECIMAL, FLOAT

Testo

VARCHAR, CHAR, TEXT

Date

DATE, TIME, DATETIME

Booleani

TRUE / FALSE

CREATE TABLE Prodotti (
 ID INT PRIMARY KEY,
 Nome VARCHAR(50),
 Prezzo DECIMAL(6,2),
 DataCreazione DATE,
 Disponibile BOOLEAN
);

Vincoli SQL

  • NOT NULL → campo obbligatorio
  • UNIQUE → valori unici
  • PRIMARY KEY → identificatore
  • CHECK → regola logica

Esempio completo

CREATE TABLE Utenti (
 ID INT PRIMARY KEY,
 Nome VARCHAR(50) NOT NULL,
 Email VARCHAR(50) UNIQUE,
 Eta INT CHECK (Eta >= 18)
);
