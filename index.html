I Vincoli (Constraints) 🛡️

I vincoli sono le "guardie del corpo" del nostro database.

  • Cosa fanno? Controllano i dati prima che vengano inseriti nella tabella.
  • Perché si usano? Per impedire l'inserimento di dati errati, incompleti o duplicati.
  • Il risultato: Dati sempre affidabili (Integrità del database).

Slide 10

Le Basi: Presenza e Unicità

⛔ NOT NULL

Obbliga a inserire un valore. Quella cella non può mai essere vuota.

Esempio: Quando ti registri a un sito, non puoi lasciare il campo "Password" vuoto.

❄️ UNIQUE

Garantisce che non ci siano cloni. Ogni valore in quella colonna deve essere diverso dagli altri.

Esempio: Due utenti non possono avere la stessa "Email".

Slide 11

Identità e Regole

🔑 PRIMARY KEY

La Chiave Primaria è la targa identificativa di ogni riga. È la fusione perfetta tra NOT NULL e UNIQUE.

Esempio: Il tuo Codice Fiscale. Ce l'hai per forza (Not Null) ed è solo tuo (Unique).

⚖️ CHECK

Un buttafuori personalizzato. Permette di inserire solo i dati che rispettano una certa condizione matematica o logica.

Esempio: Età >= 18 (Se hai 17 anni, il database ti respinge).

Slide 12

Tutto Insieme: Esempio Reale


CREATE TABLE Utenti (
    ID int PRIMARY KEY,       -- 🔑 Identificativo unico assoluto
    Nome varchar(50) NOT NULL,-- ⛔ Deve avere un nome
    Email varchar(50) UNIQUE, -- ❄️ Nessun doppione permesso
    Eta int CHECK (Eta >= 18) -- ⚖️ Solo maggiorenni
);
