Slide 2: Sintassi Base CREATE

Il comando CREATE TABLE definisce la struttura di una nuova relazione (tabella) nel database, specificando il nome e le caratteristiche fisiche dei dati.

-- Sintassi Formale
CREATE TABLE nome_tabella (
    colonna_1 tipo_dato,
    colonna_2 tipo_dato,
    ...
);
Slide 2

Slide 3: Definizione delle Colonne

  • Ogni colonna rappresenta un attributo specifico dell'entità.
  • Il dominio (tipo di dato) limita i valori inseribili per coerenza.
  • Sintassi: Nome Colonna + Spazio + Tipo + Virgola finale.
CREATE TABLE Anagrafica (
    codice_id INT,
    nominativo VARCHAR(100)
);

Slide 5: Tipi Numerici

L'efficienza dello storage dipende dalla scelta del numero:

  • INT: Interi a 32-bit (range ±2 miliardi).
  • DECIMAL(p,s): Precisione fissa per valuta e finanza.
  • FLOAT / DOUBLE: Precisione variabile per calcoli scientifici.
Slide 5

Slide 6: Stringhe e Testi

Gestione dei dati alfanumerici:

  • VARCHAR(n): Lunghezza variabile (occupa solo ciò che scrivi).
  • CHAR(n): Lunghezza fissa (ideale per codici di stato).
  • TEXT: Memorizzazione di blocchi di testo massivi.
Slide 6

Slide 7: Date e Booleani

Dati temporali e flag logici:

  • DATE: Gestisce solo Anno, Mese e Giorno.
  • TIMESTAMP: Registra istanti precisi con data e ora.
  • BOOLEAN: Supporta solo 1 (TRUE) o 0 (FALSE).

Slide 9: Vincoli di Integrità

I vincoli assicurano che i dati rispettino le regole di business prima di essere salvati permanentemente.

  • Garantiscono l'accuratezza dei dati nel tempo.
  • Evitano la ridondanza o l'incoerenza logica.
Slide 9

Slide 10: PK, NOT NULL, UNIQUE

PRIMARY KEY: L'ancora della tabella. Unica e mai nulla.

NOT NULL: Obbliga la compilazione del campo (es. Cognome).

UNIQUE: Impedisce valori uguali (es. Codice Fiscale).

Slide 10

Slide 11: Il vincolo CHECK

Il CHECK permette di definire una condizione personalizzata di validità.

-- Controllo età minima
eta INT CHECK (eta >= 18)

-- Controllo range voti
voto INT CHECK (voto >= 1 AND voto <= 10)
Slide 11

Slide 12: Esempio e Esercizio

CREATE TABLE Clienti (
    id INT PRIMARY KEY,
    email VARCHAR(100) UNIQUE NOT NULL,
    eta INT CHECK (eta >= 18)
);

Slide 13: ALTER TABLE - Evoluzione

Permette di modificare tabelle esistenti senza doverle distruggere e ricostruire.

-- Nuova colonna
ALTER TABLE Studenti ADD telefono VARCHAR(20);

-- Cambio tipo dato
ALTER TABLE Studenti ALTER COLUMN nome TYPE VARCHAR(150);
Slide 13

Slide 14: DROP COLUMN e Constraint

Raffinamento della struttura in fase di manutenzione:

-- Rimuovere colonna obsoleta
ALTER TABLE Studenti DROP COLUMN media_voti;

-- Aggiungere vincolo post-creazione
ALTER TABLE Studenti ADD UNIQUE (email);
Slide 14

Slide 15: DROP TABLE - Distruzione

OPERAZIONE CRITICA: Non esiste "Cestino".

Il comando DROP TABLE cancella lo schema e tutti i dati in esso contenuti istantaneamente.

DROP TABLE Studenti; -- Tabella eliminata
